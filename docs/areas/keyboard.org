** Vial And QMK - Accepted
:PROPERTIES:
:CUSTOM_ID: vial-and-qmk-incomplete
:END:
Vial 0.7.5, QMK 1.2.0, and =qmk-udev-rules= are present in checkpoint
generation 856, but its rules package is inert there and the device workflow is
not accepted.

Confirmed defect:

- =qmk-udev-rules= is only listed in =environment.systemPackages=.
- =hardware.keyboard.qmk.enable= evaluates to false.
- =services.udev.packages= does not contain the rules package.
- =/etc/udev/rules.d/50-qmk.rules= is absent, and =udevadm test= confirms that
  systemd-udevd does not load the package's rules from the system profile.

This explains why merely installing Vial/QMK did not grant non-root device
access.  No compatible external device was connected during the audit, so the
hardware path remains untested.

The first maintenance slice on =work/nixos-maintenance= now:

- enables =hardware.keyboard.qmk.enable= only in =hosts/msi/default.nix=;
- removes explicit =qmk-udev-rules= package ownership while retaining Vial and
  the QMK CLI;
- evaluates with QMK enabled for MSI and disabled for the AMD bootstrap;
- generates =/etc/udev/rules.d/50-qmk.rules= and the =plugdev= group;
- builds successfully without activating generation 856.

The official rule grants active-seat =uaccess= to =hidraw*= and does not add the
user permanently to =plugdev=.  This is broader than a device-specific VID:PID
rule, so keep that security boundary explicit.

No external USB HID keyboard was connected during the candidate audit.  The
only live hidraw node was the internal Synaptics touchpad, so the intended
device, VID:PID, ACL, and Vial discovery remain untested.

Activation and device acceptance passed in generation 857:

- the keyboard is foostan Corne v4 =4653:0004=, serial
  =vial:f64c2b3c=;
- its two interfaces use =/dev/hidraw1= and =/dev/hidraw2=;
- both nodes are =0660 root:plugdev= with explicit =rw= ACLs for
  =skydive420dz= and =uaccess=/=udev-acl= tags;
- Vial 0.7.5 runs from the Nix store and the user confirmed discovery;
- no firmware write or flash occurred.

Generation 856 remains the booted fallback until reboot.  Decide separately
whether the full QMK toolchain belongs system-wide or in an on-demand firmware
project shell.

** NixOS Kanata Investigation
:PROPERTIES:
:CUSTOM_ID: nixos-kanata-investigation
:END:
Goal: keep the MSI Bravo 15 C7V keyboard behavior documented while Kanata replaces keyd.

Laptop keyboard spec from MSI: blue keyboard with anti-ghost key, 99-key layout. Anti-ghosting helps with simultaneous key presses, but Linux/Kanata still only can remap keys that emit visible input events.

2026-06-05 follow-up: Kanata can remain =active (running)= after a device
hotplug/re-enumeration while its =/run/kanata-*/...= symlink still points to
the old =/dev/input/event*= node. The DrunkDeer instance hit this after the
keyboard disappeared and returned as a new event device; restarting
=kanata-internal.service= and =kanata-drunkdeer.service= recovered it. Later
investigate the trigger and decide whether a udev/systemd restart hook is worth
adding.

These are intentional machine-specific choices. Keep them documented so they can move into host-level Nix options later instead of becoming mystery strings.

| Area                                    | Scope                          | Where                                                        | Decision                                                                                         | Why                                                                                                                              | Future cleanup                                                  |
|-----------------------------------------+--------------------------------+--------------------------------------------------------------+--------------------------------------------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------+-----------------------------------------------------------------|
| Touchpad toggle                         | MSI Bravo 15 C7V               | =config/hypr/hyprland.lua=                                   | Bind =code:202= to an inline Lua toggle                                                          | MSI emits =XF86TouchpadToggle=, but Hypr did not catch the keysym bind; keycode binding worked                                   | Move keycode/device into host options                           |
| Touchpad device                         | MSI Bravo 15 C7V               | =config/hypr/hyprland.lua=                                   | Hardcode =pnp0c50:0b-06cb:7e7e-touchpad=                                                         | Hypr runtime toggle needs the exact device name from =hyprctl devices=                                                           | Move to =mkOption= per host                                     |
| Play/pause media key                    | NixOS shell                    | =config/hypr/hyprland.lua=, =config/quickshell/root/Bar.qml= | Bind =XF86AudioPlay= to Quickshell IPC                                                           | Quickshell already owns MPRIS state/control, so no extra media package is needed                                                 | Keep as-is unless media ownership changes                       |
| Extra =|= key                           | MSI Bravo 15 C7V 99-key layout | =system/modules/input.nix=                                   | Map Kanata =IntlBackslash = bksl=                                                                | Built-in 99-key layout emitted the wrong logical key for the physical =|= key                                                    | Keep with host keyboard options later                           |
| DrunkDeer A75 Pro ANSI temporary Kanata | External USB keyboard          | =system/modules/input.nix=                                   | Add separate =kanata-drunkdeer.service= with the same Caps tap/hold and Caps+Y/P clipboard layer | Makes the keyboard usable while the standalone =drunkdeerctl= flake grows into the real configuration tool                       | Remove this block once DrunkDeer firmware/app workflow is ready |
| DrunkDeer brightness layer              | External USB keyboard          | =system/modules/input.nix=                                   | Map DrunkDeer =pgup=/=pgdn= output to Kanata =brup=/=brdn=                                       | DrunkDeer Antler does not expose brightness keys for firmware remapping; the board's Fn layer can emit Page Up/Page Down instead | Revisit if we later need plain Page Up/Page Down on the board   |
| Display-mode key                        | MSI Bravo 15 C7V               | none                                                         | Leave =Fn+F11= unmapped                                                                          | It emits =Super+p=, not a clean display XF86 key; no current workflow needs it                                                   | Design explicit monitor profile/picker later                    |

** Future Kanata Layer Ideas
:PROPERTIES:
:CUSTOM_ID: future-kanata-layer-ideas
:END:
Do not rush these. The current Kanata config is intentionally small and stable. Only add layer behavior when it removes real friction and does not fight app-native bindings.

| Idea                    | Candidate behavior                                       | Notes                                                                                  |
|-------------------------+----------------------------------------------------------+----------------------------------------------------------------------------------------|
| Vim-style navigation    | Layer + =h/j/k/l= sends arrows                           | Useful outside terminal/editor contexts                                                |
| Word motion             | Layer + =w/b/e= sends word-forward/back behavior         | Needs app-safe modifier choices per platform                                           |
| Copy/paste holds        | Hold =y= for copy, hold =p= for paste                    | Candidate replacement for explicit layer =Y/P=; test carefully to avoid typing latency |
| Workspace/tag switching | Hold number keys to switch workspaces/tags               | Mirrors the Hypr workspace mental model                                                |
| Move to workspace/tag   | Shift + hold number keys to move window to workspace/tag | Should map to Hypr actions, not app-level number input                                 |
