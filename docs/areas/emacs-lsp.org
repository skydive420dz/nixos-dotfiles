** Locked Model Boundary
:PROPERTIES:
:CUSTOM_ID: locked-model-boundary
:END:
The global LSP model is locked unless an extreme language-specific constraint is
identified, documented, and accepted before implementation. Future language
slices should not rewrite the common editor model.

Global and reusable:

- Completion frontend: Company.
- Completion backend shape: =company-capf=, =company-yasnippet=,
  =company-files=, =company-keywords=, =company-dabbrev-code=.
- Completion navigation: =C-j=, =C-k=, =C-l=, =C-h=.
- Docs display policy: full LSP hover, helper-only Eldoc display.
- Code actions: =SPC c a=.
- Definition/references/type/implementation wrappers.
- Diagnostics list behavior.
- Formatting contract: use =sk-format=, not ad hoc LSP formatting, unless
  explicitly chosen.
- Helper window/display rules.
- Acceptance checklist.

Language-specific and mutable per slice:

- Major modes owned by =lsp-mode=.
- LSP server choice.
- Server command/package.
- Server settings.
- Runtime/library paths.
- Known globals.
- Keyword table mapping, when needed.
- Snippet availability.
- Formatter command under =sk-format=.
- Unsupported capabilities, such as missing =typeDefinition= support.
- Acceptance probes for that language.

** Completion Ownership
:PROPERTIES:
:CUSTOM_ID: completion-ownership
:END:
Company is the in-buffer completion frontend for Powerhouse IDE proof buffers.
LSP is the semantic source, not the whole completion system.

For lsp-mode proof buffers, source ownership is:

- =company-capf=: semantic completion from =lsp-completion-at-point=, including
  server-provided snippets when the server provides them.
- =company-yasnippet=: repo/local Yasnippet templates when available.
- =company-files=: file/path references.
- =company-keywords=: maintained language keyword fallback.
- =company-dabbrev-code=: local buffer/project words.

Do not implement a custom syntax backend while an existing maintained backend
or the language server can own the job. A custom backend requires an explicit
proposal and approval before code changes.

** Target Backend Shape
:PROPERTIES:
:CUSTOM_ID: target-backend-shape
:END:
lsp-mode proof buffers should use one shared Company backend stack:

#+begin_src emacs-lisp
(setq-local company-backends
            '((company-capf
               company-yasnippet
               company-files
               company-keywords
               company-dabbrev-code)))
#+end_src

Language-specific keyword table mappings are allowed only when needed. For
=lua-ts-mode=, =company-keywords= reuses the maintained =lua-mode= keyword
table:

#+begin_src emacs-lisp
(with-eval-after-load 'company-keywords
  (add-to-list 'company-keywords-alist '(lua-ts-mode . lua-mode)))
#+end_src

** CAPF Boundary
:PROPERTIES:
:CUSTOM_ID: capf-boundary
:END:
For lsp-mode proof buffers, CAPF stays semantic:

#+begin_src emacs-lisp
(setq-local completion-at-point-functions
            '(lsp-completion-at-point))
#+end_src

Company owns source orchestration. Do not hide snippets, keywords, paths, and
dabbrev inside CAPF unless a measured backend limitation requires it and the
tradeoff is documented before implementation.

** Runtime Rules
:PROPERTIES:
:CUSTOM_ID: runtime-rules
:END:
lsp-mode proof buffers should apply these local rules:

#+begin_src emacs-lisp
(setq-local company-minimum-prefix-length 0)
(setq-local lsp-completion-no-cache t)
(setq-local eldoc-display-functions '(eldoc-display-in-buffer))
(setq-local company-backends sk/company-code-backends)
(company-mode 1)
#+end_src

Rationale:

- =company-minimum-prefix-length= allows trigger-character completions such as
  =vim.= to open without requiring an extra typed character.
- =lsp-completion-no-cache= prevents stale server result pages from being
  filtered locally after the prefix changes.
- =eldoc-display-functions= keeps full hover docs in the helper buffer instead
  of duplicating them in the minibuffer.
- =sk/company-code-backends= is the single shared backend definition for the
  current proof stack.

** Acceptance Checklist
:PROPERTIES:
:CUSTOM_ID: acceptance-checklist
:END:
Language slices are not accepted because one server starts or one completion
source responds. A language is accepted only when it works as part of the shared
editor system.

Each lsp-mode language slice must prove:

- LSP semantic completion works.
- Server snippets or Yasnippet templates work when available.
- Keyword completion works through the server or =company-keywords= when
  available.
- File/path completion works.
- Local/project word completion works.
- Hover/docs work, including explicit doc commands.
- Hover/docs show full LSP hover content in the helper buffer without duplicate
  minibuffer output.
- Definition and references work where the server supports them.
- Diagnostics work through Flycheck.
- Code actions work where LuaLS supports them.
- Formatting uses the =sk-format= contract.
- Completion navigation works with =C-j=, =C-k=, =C-l=, and =C-h=.
- No hardcoded machine paths.
- No hidden one-off package hacks.
- No unrelated keymap changes.
