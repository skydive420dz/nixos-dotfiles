* Emacs
:PROPERTIES:
:CUSTOM_ID: emacs
:END:
This is a serious Doom Emacs experiment, not a Neovim replacement plan.

** Shape
:PROPERTIES:
:CUSTOM_ID: shape
:END:
- Nix installs Emacs and the support tools.
- Doom's live config lives in [[../../config/doom][config/doom]].
- Home Manager links =~/.config/doom= and =~/.doom.d= to the repo config so editing =init.el=, =config.el=, or Org notes does not require a rebuild.
- Doom itself is cloned from upstream with =doom-bootstrap=, into =~/.config/emacs=.
- NixOS runs Emacs as a user daemon; graphical launches use =emacsclient= through the =doom-emacs= wrapper.

** Installed By Nix
:PROPERTIES:
:CUSTOM_ID: installed-by-nix
:END:
- =emacs-pgtk= from the flake's Nixpkgs pin.
- =doom-emacs= opens a graphical client frame against the daemon.
- =doom-emacs-terminal= opens a terminal client frame against the daemon.
- Core project tools: Git, ripgrep, fd, EditorConfig, tree-sitter.
- Rich document support: sqlite, gnutls, ImageMagick, zstd, unzip, pandoc, TeX Live Small, aspell English, and Harper.
- vterm native support: libvterm, cmake, gnumake, gcc, pkg-config.
- Nix editing support: nil and nixfmt.
- Lua, QML/Qt, Rust, C, and C++ LSP support: lua-language-server, qt6.qtlanguageserver, rust-analyzer, and clang-tools.

** Doom Modules
:PROPERTIES:
:CUSTOM_ID: doom-modules
:END:
- Evil mode for Vim muscle memory.
- Org mode for notes and project planning.
- Magit for Git.
- Dired for file management.
- vterm for the embedded terminal.
- Vertico/company, LSP, snippets, syntax/spell checking, Markdown, Nix, shell, Lua, QML/Qt, Rust, C/C++, JSON, YAML, and Emacs Lisp.
- Harper runs through LSP for Org, Markdown, and plain text diagnostics while writing.

** Local Behavior
:PROPERTIES:
:CUSTOM_ID: local-behavior
:END:
- =config/doom/config.org= is the readable source of truth. =config.el= is kept in sync from it for Doom.
- =C-h=, =C-j=, =C-k=, and =C-l= move focus between Emacs windows, including inside vterm insert state.
- Doom's native =SPC w ...= window map is intentionally left to Doom. Do not override =SPC w H/J/K/L= with custom swap/move helpers unless we explicitly decide to replace Doom's defaults.
- Line numbers are relative in code buffers and disabled in terminal buffers.
- Spellcheck uses repo-local =config/doom/spell/en_US.pws= for personal words.
- QML snippets live in =config/doom/snippets/qml-mode=.
- Sky themes are native Doom themes in =config/doom/themes/=, backed by the global selector output in =~/.config/theme/current/emacs-theme.el=.
- =SPC h r t= reloads the active Sky theme after =theme-select= changes.
- The desktop launcher entry named =Emacs= opens =doom-emacs=, so closing the window only closes that frame. The daemon keeps buffers, projects, LSP state, and Org files alive until the user service is restarted or stopped.

** Commands
:PROPERTIES:
:CUSTOM_ID: commands
:END:
After rebuilding once:

#+begin_src sh
doom-bootstrap
#+end_src

Then use:

#+begin_src sh
doom sync
doom-emacs
#+end_src

Run =doom sync= after changing =config/doom/init.el= or =config/doom/packages.el=. Most edits to =config.el= and Org files are live after restarting or reloading Emacs.

Daemon/service commands:

#+begin_src sh
systemctl --user status emacs.service
systemctl --user restart emacs.service
systemctl --user stop emacs.service
doom-emacs
doom-emacs-terminal
#+end_src

=SPC h r r= reloads Doom config into the current running daemon. It is useful for small Elisp edits, but it is not a clean process restart. Use =doom sync= plus =systemctl --user restart emacs.service= after changing packages/modules or when the daemon has loaded bad state.
