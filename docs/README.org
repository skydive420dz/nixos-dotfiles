* How The Documentation Works
Start with [[../README.org][the repository README]].  Project and area notes
describe durable ownership and direction.  Handoffs record current execution
state.  Audits are dated evidence, not implementation authority.

Documentation does not turn a build into activation and does not authorize
commits, pushes, service changes, or branch deletion.

* Writing Rules
- Use =LIVE=, =CANDIDATE=, =IN PROGRESS=, =ACCEPTED=, and =HISTORICAL=
  deliberately.
- Keep volatile state in the active handoff instead of copying it into every
  area note.
- Keep current ownership near the top and detailed history below it.
- Prefer links and concise decision records over repeated roadmaps.
- Keep build, activation, runtime acceptance, promotion, and cleanup as separate
  facts.
