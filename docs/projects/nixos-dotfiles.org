* Current System
:PROPERTIES:
:CUSTOM_ID: current-system
:END:
The MSI laptop is the primary live host and uses the =#nixos= flake output.
The =#desktop-amd= output is a guarded bootstrap target whose machine-specific
hardware acceptance is still future work.

For volatile branch, generation, output, build, activation, and promotion state,
use [[../handoffs/2026-07-18-main-checkpoint.org][the active checkpoint handoff]].

Durable ownership:

- Hyprland behavior lives in =config/hypr/hyprland.lua=.
- Quickshell owns the desktop shell surfaces under =config/quickshell/=.
- =scripts/theme-select= and =theme/styles.json= own global theme state.
- Clean Emacs is the active Emacs configuration; Doom is retired.
- NVF owns Neovim packaging, plugins, and LSP while =config/nvim/= owns live Sky
  behavior.
- Ollama is retired from the intended system configuration.
- Vial/QMK device integration and a Linux VS Code port are unfinished work, not
  accepted current features.

* Entry Points
:PROPERTIES:
:CUSTOM_ID: entry-points
:END:
- =flake.nix=: host outputs and shared special arguments.
- =hosts/msi/=: current MSI host entrypoint.
- =hosts/desktop-amd/=: AMD bootstrap entrypoint.
- =system/modules/=: reusable NixOS modules.
- =home/modules/=: Home Manager modules and live-link ownership.
- =config/=: application-native configuration.
- [[../areas/modules.org][Module and host structure]]

* Normal Commands
:PROPERTIES:
:CUSTOM_ID: normal-commands
:END:
#+begin_src sh
git status --short --branch

nix --extra-experimental-features 'nix-command flakes' \
  build --print-out-paths \
  '.#nixosConfigurations."nixos".config.system.build.toplevel' \
  --no-link --impure --no-write-lock-file --show-trace

nrs
hyr reload
systemctl --user restart quickshell
#+end_src

The standalone build does not activate the machine.  =nrs= is the separate
interactive activation command.

* Direction
:PROPERTIES:
:CUSTOM_ID: direction
:END:
- Keep =main= as the checkpoint lane.
- Work in small, measurable slices with rollback.
- Preserve the intentional live-link workflow and validate resolved live bytes.
- Move machine facts into =hosts/<host>/= without hiding reusable mechanisms.
- Fix confirmed defects before performance experiments.
- Keep Vial/QMK, Emacs TLC, VS Code, Quickshell optimization, and GC as separate
  work streams.

Current post-checkpoint lanes:

- =work/nixos-maintenance=: system correctness, Vial/QMK permissions, security,
  ownership, organization, and conservative GC.
- =work/quickshell-cleanup=: deterministic shell bugs, service lifecycle,
  backend consolidation, then measured optimization.
- Emacs: finish the active language acceptance decision, then optimize only
  observed daily-use friction.
- VS Code: adapt the macOS Home Manager configuration for Linux rather than
  importing Darwin-specific extension pins.
