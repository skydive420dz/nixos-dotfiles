** Current
:PROPERTIES:
:CUSTOM_ID: current
:END:
- Current live behavior is script-backed: the Quickshell launcher button opens =scripts/launcher-toggle= on left click and =scripts/clipboard-toggle= on right click.
- =scripts/launcher-toggle= uses =fuzzel --show drun= with the generated Sky theme config.
- =scripts/clipboard-toggle= uses =cliphist list | fuzzel --dmenu= and decodes the selected item back to =wl-copy=.
- The old dedicated Quickshell overlay app is not in the active tree right now. Resurrection should start from history/reference branches and port only the pieces that still fit the current structure.

** Decisions
:PROPERTIES:
:CUSTOM_ID: decisions
:END:
- Keep launcher and clipboard as separate tools until focus restoration, keyboard navigation, and close behavior are boring and reliable.
- Keep =fuzzel= scripts as the working fallback until the native Quickshell app can fully replace them.
- Use Quickshell screen binding for layer windows.
- Follow the Quickshell ownership rules: =root/= owns surfaces, =modules/<name>/= owns behavior, =common/= owns reusable primitives only.
- Avoid full-screen invisible masks unless the overlay is intentionally click-out-to-close and the mask is verified not to steal unrelated input.
