#+title: NixOS Dotfiles History And Decision Log
#+startup: overview

* Purpose
This is the short human memory log for major direction changes.  It preserves
context without turning the root README into a changelog.  Current instructions
live in the root README, area contracts, and the active handoff; dated audits
remain the detailed evidence.

* 2026-05 - Desktop Ownership Settles
- Hyprland moved to a Lua-first configuration.
- Quickshell became the native owner of the bar, launcher, clipboard, wallpaper,
  OSD, tray, media, and status surfaces.
- The native launcher and clipboard replaced their Fuzzel fallbacks.
- Quickshell replaced =awww= as wallpaper owner.
- The Sky theme registry and =scripts/theme-select= became the global theme
  authority.

* 2026-06 - Clean Emacs Becomes Primary
- Clean Emacs replaced Doom as the active editor configuration.
- Useful behavior, bindings, theme faces, Org, terminal, and language tooling
  were migrated deliberately.
- Doom remained briefly as frozen reference material while parity was checked.
- Emacs language work later moved to a hybrid model: accepted Lua and Nix
  =lsp-mode=/Company slices, an in-progress Python slice, and Eglot/Corfu for
  languages not yet migrated.

* 2026-07 - Audit And Checkpoint
- The live MSI system and Quickshell runtime were audited against the actual
  machine and linked configuration.
- The audits identified confirmed correctness, lifecycle, security, ownership,
  and hygiene issues and separated them from performance hypotheses.
- A Nixpkgs/NVF/Hyprland/Quickshell input update exposed the removed NVF
  =vim.languages.rust.lsp.package= option.  The checkpoint candidate removes
  only that obsolete override.
- Doom was explicitly retired rather than kept as a second configuration.
- Ollama was explicitly retired; model-state deletion remains ordered after the
  running service is removed.
- The user designated the successful full no-link build as the source/build
  checkpoint.  Commit =4eded33= was fast-forwarded to =main= without changing
  the live generation.  Activation and runtime evidence remain separate facts.
- Generation 856 later activated and booted the verified output.  Ollama's unit,
  binary, and listener are gone.  The user then completed privileged model-state
  removal; visual hand tests remain a separate acceptance gate.
- The user accepted generation 856 after the remaining hand tests.  That closes
  the checkpoint gate and permits tag-first legacy branch archival and fresh
  post-checkpoint work lanes.
- Eleven legacy branch tips were preserved as verified remote annotated tags,
  then their local and remote branch names were retired.  Fresh maintenance and
  Quickshell lanes were created from accepted =main=.
- The first maintenance slice enabled MSI-scoped QMK udev ownership.  Generation
  857 granted active-seat ACLs to the connected Corne v4, and Vial discovered
  the keyboard without a firmware write.
- The user explicitly retained QMK CLI as a system-wide host capability; its
  package and closure cost are intentional rather than pending cleanup.

* Unfinished Experiments Preserved As Direction
- Vial and QMK packages initially reached the MSI system without active QMK
  udev integration or device acceptance.  The focused generation 857 slice
  resolved that defect; QMK CLI remains system-wide by explicit decision.
- The NixOS VS Code placeholder never became a configuration.  A future port
  should adapt the macOS Home Manager module for Linux.
- Emacs optimization remains real-use-driven work, not a broad rewrite.
- Quickshell optimization follows correctness and service lifecycle repairs.
