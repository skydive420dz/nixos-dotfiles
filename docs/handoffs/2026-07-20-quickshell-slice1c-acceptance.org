* Accepted Implementation Boundary

Slice 1C makes four narrow behavior changes:

1. Wallpaper state load and directory listing use direct =Process.command=
   argument arrays instead of dynamic =bash -lc= strings.
2. The module-private =wallpaper-io= helper receives all dynamic values as
   positional arguments.  =JSON.stringify= remains only JSON serialization.
3. State and video thumbnails are written to unique same-directory temporary
   files and renamed only after validation or successful FFmpeg completion.
   State publication precedes the in-place watched signal update.
4. A thumbnail card reloads only after helper exit zero and only when the
   completed wallpaper/output identity still matches that card.

The checker requires the helper, its executable bit, Bash syntax, and targeted
ShellCheck without making unrelated legacy scripts part of a new ShellCheck
cleanup.
