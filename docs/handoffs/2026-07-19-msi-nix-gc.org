age-based Nix garbage-collection timer in an explicitly unarmed state and

first actual GC, store optimization, and boot-entry retention remain separate

* Confirmed Defect
=home/modules/programs/shell-aliases.nix= defined:

#+begin_src sh
ncg='sudo nix-collect-garbage -d'
#+end_src

The =-d= operation deletes older profile generations before collecting, which
can erase expected rollback entries.  The candidate removes the alias.  It does
not replace it with another unprompted maintenance command.  A future manual
alias can be considered only after the timer policy and first observed run are
accepted.

The alias table is shared by the Home Manager outputs for both flake hosts, so
the safe alias removal is shared too.  Only the automatic timer policy is
MSI-specific.

* Candidate Policy
The policy lives in =hosts/msi/maintenance.nix= and is imported only by the MSI
host:

#+begin_src nix
{ lib, ... }:

{
  nix.gc = {
    automatic = true;
    dates = "Sun *-*-* 04:15:00";
    options = "--delete-older-than 30d";
    persistent = false;
    randomizedDelaySec = "45min";
  };

  systemd.timers.nix-gc.wantedBy = lib.mkForce [ ];
}
#+end_src

The timer's first nominal calendar event after authoring is Sunday, July 26,
2026 at 04:15 EDT with up to 45 minutes of randomized delay.  Normal systemd
timer accuracy also applies, so this is not a guaranteed exact endpoint.
=persistent = false= prevents activation from catching up the already-passed
July 19 window.

The =wantedBy= override is the first-run safety latch: the unit is built and can
be inspected, but activation does not enable or start it and no calendar event
is armed.  After the first GC is separately authorized and observed, removing
this override is its own source, build, and activation slice.

The pinned =nix-collect-garbage= manual confirms that
=--delete-older-than 30d= first applies age-based generation deletion to
multiple profiles it discovers in default, system, and per-user profile
locations.  It is not restricted to the NixOS system profile and can affect
profiles belonging to other users.  The live inventory currently found:

- the 11 NixOS system generations listed above;
- one current Home Manager GC root and no independent Home Manager generation
  history;
- one current root-channel generation;
- one current user-channel generation;
- no standalone user =profile-N-link= history.

The profile inventory must be repeated immediately before the first authorized
GC.  A newly discovered user or Home Manager history pauses that maintenance
gate until its retention ownership is decided.

This candidate intentionally excludes:

- =nix.optimise= and =auto-optimise-store=;
- a Lanzaboote or systemd-boot configuration limit;
- cleanup of dangling GC roots or temporary roots;
- cleanup of legacy boot entries, channels, caches, journals, or coredumps;
- automatic flake or system upgrades.
