* Source Decision
The narrow correction is:

1. Set =hardware.keyboard.qmk.enable = true= in =hosts/msi/default.nix=.
2. Remove =qmk-udev-rules= from =environment.systemPackages=.
3. Keep Vial 0.7.5 and QMK CLI 1.2.0 installed.
4. Keep =hardware.keyboard.qmk.keychronSupport = false= unless the physical
   device proves that Keychron-specific rules are required.

MSI owns the enable because the AMD bootstrap imports the shared
=system/modules= tree and must remain unchanged.  A new keyboard module and
movement of existing Kanata paths are separate organization work.

* NixOS Behavior And Security Boundary
The pinned NixOS QMK module:

- adds =qmk-udev-rules-0.27.13= to =services.udev.packages=;
- creates =plugdev=;
- does not install Vial or the QMK CLI;
- does not add the user permanently to =plugdev=.

The official generic hidraw rule is:

#+begin_example
KERNEL=="hidraw*", MODE="0660", GROUP="plugdev", TAG+="uaccess", TAG+="udev-acl"
#+end_example

Local graphical access relies on logind's active-seat =uaccess= ACL.  The
generic rule affects all hidraw devices at the active seat, not only one
keyboard; that is the upstream QMK policy and an explicit security tradeoff.
